# Классификация новостей Lenta.ru (100 000 документов).

Этот проект реализует полный воспроизводимый NLP‑пайплайн для классификации русскоязычных новостных статей по темам с использованием классических методов машинного обучения. Включает загрузку данных, предобработку текста, векторизацию, обучение модели, подбор гиперпараметров, оценку качества и анализ ошибок.

## 1. Обзор проекта.

Репозиторий содержит end‑to‑end пайплайн классификации текстов, построенный на подвыборке из 100 000 документов корпуса новостей Lenta.ru. 
Цель — оценить, насколько далеко можно продвинуться с классическими методами NLP (CountVectorizer, TF‑IDF, Logistic Regression) на большом реальном русскоязычном датасете.

Пайплайн включает:

 - эффективную предобработку русского текста (нормализация + лемматизация);
 - стратифицированное разбиение (60 / 20 / 20);
 - базовую оценку качества (DummyClassifier);
 - две стратегии векторизации (Count и TF‑IDF);
 - подбор гиперпараметров через GridSearchCV;
 - финальную оценку на отложенной выборке;
 - анализ ошибок и выводы.

## 2. Датасет.

 - источник: датасет новостей Lenta.ru (Corus / Kaggle);
 - использовано: 100 000 документов;
 - поля: title, text, topic;
 - редкие классы удалены для корректной стратификации.

Пример записи:  " Каждая строка датасета представляет собой одну новость и содержит три поля: title, text и topic".

## 3. Пайплайн предобработки.

Этапы предобработки:

 - приведение текста к нижнему регистру;
 - удаление небуквенных символов;
 - нормализация пробелов;
 - лемматизация с помощью pymorphy3;
 - удаление пустых текстов;
 - фильтрация классов с количеством примеров < 2;
 - создание очищенного столбца text_clean.

Почему лемматизация?

Лемматизация снижает разреженность признаков и повышает качество классификации, что особенно важно для языков с богатой морфологией, таких как русский.

## 4. Разделение на выборки.

 - обучение: 60 %;
 - валидация: 20 %;
 - тест: 20 %;
 - стратификация по topic;
 - фиксированный random_state = 42 для воспроизводимости.

## 5. Бейзлайн.

 - DummyClassifier(strategy="most_frequent");
 - точность на валидации: 0.3849;
 - macro F1: 0.061.

Этот результат задаёт минимальный уровень качества для сравнения.

## 6. Модели и векторизация

Оценивались два пайплайна: 

 - CountVectorizer + LogisticRegression;
 - TfidfVectorizer + LogisticRegression

Подбираемые гиперпараметры.

Параметры векторизатора:

 - max_features: 20 000, 50 000;
 - ngram_range: (1, 1), (1, 2);

Параметры Logistic Regression:

 - C: 0.5, 1.0, 2.0;

Параметры GridSearchCV:
 
 - 3‑кратная кросс‑валидация;
 - метрика: macro F1;
 - n_jobs = -1

Лучший результат CountVectorizer:

{'clf__C': 0.5, 'vect__max_features': 50000, 'vect__ngram_range': (1, 1)}  

*Средний macro F1 = 0.6507*

Лучший результат TF‑IDF:

*Средний macro F1 = 0.6465*

CountVectorizer показал немного лучшее качество.

Что можно было бы тюнить в продакшн‑пайплайне? 

Если бы проект развивался в сторону промышленного решения, можно было бы расширить сетку гиперпараметров и улучшить пайплайн.

Дополнительные параметры векторизатора:
 - min_df — игнорировать очень редкие слова (уменьшение шума);
 - max_df — игнорировать слишком частые слова (аналог стоп‑слов);
 - stop_words — собственный список стоп‑слов;
 - sublinear_tf=True — логарифмирование частот;
 - tokenizer / preprocessor — кастомная токенизация и предобработка.

Дополнительные параметры модели:
 - class_weight="balanced" — улучшение качества на редких классах;
 - penalty="l1" или "l2" — тип регуляризации;
 - solver="liblinear" или "saga" — оптимизированные солверы для разреженных данных;
 - max_iter — увеличение числа итераций при проблемах с сходимостью.

Усиление пайплайна:
 - символьные n‑граммы — устойчивость к опечаткам и коротким текстам;
 - feature engineering — длина текста, количество чисел, наличие сущностей;
 - семантические эмбеддинги — Word2Vec, FastText, трансформеры;
 - oversampling / аугментация — улучшение качества на редких классах.

## 7. Финальная оценка (тестовая выборка).

 - Accuracy: 0.8583;
 - Macro F1: 0.6607.

Интерпретация:

 - высокая точность на частых классах;
 - более низкий recall на редких;
 - типичное поведение для несбалансированных новостных датасетов.

## 8. Анализ ошибок.

Наиболее частые ошибки классификации:

 - Мир → Россия;
 - Россия → Мир;
 - Экономика → Россия;
 - Жизнь → Мир.

Эти путаницы возникают из‑за пересечения лексики и сходства контекста между темами, что усложняет их разделение при использовании классических bag‑of‑words‑моделей.

## 9. Возможные улучшения.

 - балансировка классов (взвешивание, oversampling);
 - добавление n‑грамм и символьных n‑грамм;
 - использование LinearSVC;
 - добавление семантических признаков (Word2Vec, FastText);
 - таргетированная аугментация редких классов.

## 10. Как запустить.

```
# установка зависимостей
pip install -r requirements.txt

# обучение лучшей модели (CountVectorizer или TF-IDF)
python src/train.py

# оценка на отложенной тестовой выборке
python src/evaluate.py
```
